# Release v0.7.0

## Summary

Migrates the server from Flask (WSGI) to Quart (ASGI) and adds real-time WebSocket push (Phase G.6). Nodes connected to `GET /api/v1/ws` now receive a `block_mined` event the moment a block is mined, without polling. Also fixes `migrations/migrate.py` to auto-load `.env` and require `DATABASE_URL` explicitly, eliminating the `UnicodeDecodeError` that occurred when running without the variable exported.

## Included Changes

- Replace `Flask>=3.0` with `quart>=0.19` — all route handlers and error handlers become `async def`; rate limiter wrapper becomes async; no monkey-patching.
- Add `WebSocketHub` (`api/websocket_hub.py`) — manages active WebSocket connections via asyncio queues; `serve(send_fn)` accepts an injectable send function for testability; `broadcast(payload)` fans out to all connected clients.
- Add `GET /api/v1/ws` WebSocket endpoint — peers connect and receive `{event: block_mined, block: {...}}` in real time when any node mines a block.
- `POST /mine_block` calls `hub.broadcast()` after mining in addition to `propagator.notify_resolve()`.
- Add `pytest-asyncio>=1.0`; set `asyncio_mode = auto` in `pytest.ini`; migrate all API test files to Quart async test client (`async with app.test_client()` / `await`).
- Add `load_dotenv()` to `migrations/migrate.py`; remove hardcoded `postgres:postgres` default; exit with clear error if `DATABASE_URL` is not set.
- 9 new unit tests for `WebSocketHub`; total suite 71 tests.

## Validation

- 71 unit tests pass (`PYTHONPATH=. py -m pytest -q -m "not integration"`).
- Coverage: 81% total (gate: 80%).
- WebSocket: connect to `ws://localhost:5000/api/v1/ws`, mine a block via `POST /mine_block`, receive `block_mined` event.
- `python migrations/migrate.py` runs without error when `.env` is present; exits cleanly with an error message when `DATABASE_URL` is absent.

## Notes

- Server now requires an ASGI runner for production — `python basic-blockchain.py` uses Quart's built-in development server (Hypercorn). For production, use `hypercorn basic-blockchain:app`.
- Synchronous psycopg2 calls inside async handlers run on the event loop thread — acceptable for a simulator; for high-throughput production use, migrate to `asyncpg`.
- Phase G debt fully closed. Next: Phase H (Frontend / Enterprise / Web4 tracks).
