# Release v0.6.0

## Summary

Adds transaction propagation and block push notification (Phase G.3/G.4): nodes now broadcast new transactions to all registered peers automatically and notify peers to run consensus after each mined block. Introduces `PropagationService` in the domain layer, `PostgresNodeRegistry` for durable peer storage, migration V005, and `python-dotenv` auto-loading so the server starts correctly with or without a `.env` file.

## Included Changes

- Add `PropagationService` — `broadcast_transaction(tx)` posts to all peers via `POST /api/v1/transactions` with `X-Propagated: 1` loop-breaker header; `notify_resolve()` fires `GET /api/v1/nodes/resolve` on each peer after a block is mined.
- Add `PostgresNodeRegistry` implementing `NodeRegistryProtocol` against a `nodes (url TEXT PRIMARY KEY)` table; selected automatically by `create_app()` when `dsn` is set.
- Add V005 migration creating the `nodes` table; follows the same idempotent `CREATE TABLE IF NOT EXISTS` pattern as prior migrations.
- Wire `PropagationService` into `POST /transactions` (broadcasts unless `X-Propagated` header present) and `POST /mine_block` (notifies peers to resolve).
- Add `python-dotenv` and `load_dotenv()` at startup — `.env` is loaded automatically; `DATABASE_URL` defaults to `None` so the app starts in in-memory mode when no `.env` is present.
- Add `docs/architecture/phase-g3-g4-propagation.md` — four design decisions with rationale, component interaction diagram, and limitations table.
- 14 new unit tests; total suite 62 tests.

## Validation

- 62 unit tests pass (`PYTHONPATH=. py -m pytest -q -m "not integration"`).
- Coverage: 80% total (gate: 80%).
- Transaction broadcast and mine-block notify-resolve verified with mocked peers.
- `python basic-blockchain.py` starts in in-memory mode without `.env`; PostgreSQL mode activates automatically when `.env` is present.

## Notes

- Peer calls are synchronous and fire-and-forget — a slow or unreachable peer adds latency proportional to its timeout (default 3 s). Background dispatch is deferred to a future phase.
- No retry on broadcast failure; a peer that missed a transaction will receive it on the next `GET /nodes/resolve` cycle.
- WebSocket push and full gossip protocol are deferred to a future phase.
- See `docs/architecture/phase-g3-g4-propagation.md` for full design rationale and known limitations.
