# Release v0.8.0

## Summary

Hardening release that closes a batch of GAP findings raised after the v0.7.0 deployment of Quart + WebSocket. No new endpoints; every change is a defensive correction or a test-coverage backfill on top of the existing API surface. The goal of the release was to make every contract previously documented in `business-rules.md` actually exercised by automated tests, and to remove silent failure modes introduced by earlier exception handling.

## Included Changes

- **GAP-01 — Health unit coverage:** add unit tests for `check_db_connectivity` so the degraded-mode branch (`503` with `db: "error"`) is exercised without a live PostgreSQL.
- **GAP-02 — Decimal amount handling:** require `Transaction.amount` to round-trip through `Decimal`, preventing `float` precision drift between client request and persisted row.
- **GAP-03 — Consensus fetch coverage:** test `_fetch_chain` against the real HTTP path (mocked at the transport boundary) instead of the previous monkey-patched stub.
- **GAP-04 — Transaction persistence:** ensure the mempool table is the single source of truth for pending transactions when running with the PostgreSQL backend (no shadow in-memory caching).
- **GAP-05 — Whitespace validation:** reject sender or receiver values that are blank after `.strip()`; previously a sender of `"   "` could pass schema validation.
- **GAP-06 — Silent error logging:** replace `except: pass` in peer propagation with structured `logger.warning("peer_unreachable", …)` carrying the failed URL and exception class.
- **GAP-07 — Input length validation:** enforce a 255-character cap on `sender` and `receiver` to align with the underlying `TEXT` column expectations and mitigate amplification attacks.
- **Infrastructure fixes:**
  - `migrations/migrate.py` autoloads `.env` and exits with a clear message when `DATABASE_URL` is missing.
  - Promotion workflow no longer fails when `release/*` and `production` happen to be at the same commit (no-diff case).
  - Default DSN no longer hardcoded as `postgres:postgres`.

## Validation

- Unit suite: 87 tests pass under `PYTHONPATH=. py -m pytest -q -m "not integration"`.
- Coverage: 84% total (gate: 80%).
- Integration suite (PG): all gap-related cases pass against a clean test DB.
- WebSocket regression: `block_mined` event still fires after each `POST /mine_block`.

## Notes

- This release was tagged but its release-note file was not committed at the time. It is documented retroactively here for traceability so that v0.9.0's notes can refer to a clean baseline.
- No breaking API changes. No data-model migrations beyond what was shipped before v0.8.0.
- Phase G debt remains closed.
